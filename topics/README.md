sur les architectures backend modernes.

Contrairement au dossier `projects/`, ces documents
ne sont pas spécifiques à un projet particulier.
Ils décrivent des **patterns, pratiques et concepts
réutilisables** dans différents systèmes.

# Objectif de ces notes

Ces documents visent à :

- structurer les connaissances techniques
- documenter des patterns d’architecture
- capitaliser l’expérience acquise sur des projets réels
- servir de référence pour la conception de systèmes backend

Ils constituent progressivement un **engineering handbook**
pour les architectures backend modernes.
