# Nginx

## Sécurité

Configuration Nginx sécurisée :

- `server_tokens off`
    
- Headers de sécurité :
    
    - HSTS
        
    - X-Frame-Options
        
    - X-Content-Type-Options
        
    - Referrer-Policy
        
    - Permissions-Policy
        
    - Cross-Origin headers
        

Protection des endpoints sensibles :
/api/v1/auth  
/api/  
/api/v1/sejours

Blocage de :

/actuator

---

## Rate Limiting

Mise en place de plusieurs zones :

limit_req_zone $binary_remote_addr zone=api_general:10m rate=50r/s;  
limit_req_zone $binary_remote_addr zone=api_auth:10m rate=10r/s;  
limit_req_zone $binary_remote_addr zone=api_pagination:10m rate=30r/s;

Configuration des bursts :

limit_req zone=api_pagination burst=150 nodelay;

Objectif :

- éviter les attaques
    
- éviter les **429 Too Many Requests**
    
- supporter la pagination rapide côté frontend

## Gestion des erreurs API

Création d'une classe :

ApiError

Objectif :

- normaliser les erreurs API
    
- afficher les messages backend
    

Exemple :

429 Too Many Requests

devient :

Trop de requêtes, veuillez patienter.

au lieu de :

Erreur inconnue
